## What To Build

Create all Eloquent models, migrations, and PHP enums for the data model defined in SPEC §4.

### Enums (app/Enums/)
- `Status`: open, in_progress, resolved (backed string enum)
- `Priority`: low, medium, high, critical (backed string enum, with `needsAttention(): bool`)
- `Visibility`: private, public (backed string enum)
- `Permission`: view, comment, edit (backed string enum, with `canComment(): bool`, `canEdit(): bool` — ladderized)

### Models + Migrations
- **User** — extend existing Breeze user (no changes needed unless adding fields)
- **Issue** — all fields from SPEC §4.2. Relationships: belongsTo(User), belongsTo(Category), hasMany(Comment), hasMany(IssueShare). SoftDeletes. Casts for enums.
- **Comment** — SPEC §4.4. Relationships: belongsTo(Issue), belongsTo(User).
- **Category** — SPEC §4.3. Auto-generate slug from name in model boot. Relationship: hasMany(Issue).
- **IssueShare** — SPEC §4.5. Relationships: belongsTo(Issue), belongsTo(User). Unique constraint on (issue_id, user_id).

### Indexes
Per SPEC §4.6:
```
issues:       [user_id], [category_id], [status], [priority], [visibility]
              composite: [status, priority], [user_id, status]
comments:     [issue_id], [user_id]
issue_shares: unique [issue_id, user_id]
categories:   unique [slug]
```

### Model Accessors / Methods
- `Issue::needsAttention()`: computed from priority + deadline (see ADR-005)
- `Category`: auto slug generation on creating event
