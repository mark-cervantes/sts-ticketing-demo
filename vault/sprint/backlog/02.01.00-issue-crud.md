## What To Build

Full Issue CRUD with validation, business rules, and soft delete.

### Endpoints
| Method | Route           | Action                    |
| ------ | --------------- | ------------------------- |
| POST   | /issues         | Create issue              |
| GET    | /issues         | List (paginated, filtered)|
| GET    | /issues/{id}    | View with comments        |
| PATCH  | /issues/{id}    | Update                    |
| DELETE | /issues/{id}    | Soft delete               |

### Service Layer
**IssueService** (app/Services/IssueService.php):
- `create(User $user, array $data): Issue` — set defaults, compute needs_attention, dispatch job
- `update(Issue $issue, array $data): Issue` — validate transitions, recompute needs_attention, re-trigger summary if description changed
- `delete(Issue $issue): void` — soft delete

### Form Requests
- **StoreIssueRequest**: title (required, max:255, trimmed), description (required, trimmed), priority (required, valid enum), category_id (required, exists:categories), visibility (optional, valid enum), deadline_at (optional, after:now)
- **UpdateIssueRequest**: same fields but all optional. Includes `updated_at` for optimistic locking check.

### Controller
- Thin controller delegates to IssueService
- Applies IssuePolicy on every action
- List uses `scopeAccessibleBy($user)` + filters + pagination

### List Filtering (SPEC §5.6)
- `?status=open` — filter by status enum value
- `?priority=high` — filter by priority enum value  
- `?category=billing` — filter by category **slug** (resolve to category_id)
- All combinable. Invalid values → ignored (no 422).
- Pagination: 15 per page, `?page=2`
- Sort: `?sort=created_at&direction=desc` (default: needs_attention desc, priority desc, created_at desc)

### Business Rules
- Create: status=open, visibility=private, summary_status=pending (defaults)
- Create: compute needs_attention immediately
- Create: dispatch GenerateSummaryJob
- Update description: reset summary_status=pending, dispatch new GenerateSummaryJob
- Update status only: do NOT re-trigger summary
- Update priority: recompute needs_attention
- Optimistic locking: client sends updated_at, mismatch → 409

### Response Shape
Issue response includes:
- All issue fields
- `category`: { id, name, slug } (not just category_id)
- `user`: { id, name } (owner)
- `comments_count` on list (not full comments)
- `comments` with `comments.user` on single view
