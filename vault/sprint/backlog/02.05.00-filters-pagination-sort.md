## What To Build

Enhance the issue list endpoint with proper filtering, pagination, and sorting.

### Filtering (SPEC §5.6)
Query params — all optional, combinable:
- `?status=open` — exact match on status enum
- `?priority=high` — exact match on priority enum
- `?category=billing` — match on category **slug** (resolve via Category::where('slug', ...))
- Invalid values → ignored silently (no 422)

### Pagination
- Default 15 per page
- `?per_page=25` (optional, max 50)
- `?page=2`
- Response includes pagination metadata (total, current_page, last_page, etc.)

### Sorting
- `?sort=created_at` (default)
- Allowed sort fields: created_at, updated_at, priority, deadline_at
- `?direction=desc` (default: desc)
- Default sort: needs_attention desc, then priority desc, then created_at desc

### Implementation
- Build query scopes on Issue model: `scopeFilterByStatus`, `scopeFilterByPriority`, `scopeFilterByCategory`
- Chain scopes in controller/service
- Use Eloquent `paginate()` with `appends()` to preserve filter params in pagination links

## Tests Required

Feature:
- Filter by single status → correct results
- Filter by single priority → correct results

- Combined all three → narrowed results
