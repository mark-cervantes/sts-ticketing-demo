# Task 03.02.00: Kanban Board with Drag-Drop Status Transitions

## Business Requirements

### Problem
Users need a visual Kanban board to see all issues organized by status (Open, In Progress, Resolved) and move them between columns via drag-and-drop.

### User Story
As a user, I want to see my issues in a Kanban board with drag-and-drop between columns, so I can quickly triage and update issue status.

### Acceptance Criteria
- [ ] Three columns: Open, In Progress, Resolved (from Status enum)
- [ ] Issues fetched from `GET /api/issues` with filters applied from sidebar
- [ ] Issue cards show: title, priority badge, needs_attention flag, category, summary preview (truncated), deadline
- [ ] Cards sortable within column: needs_attention first → priority desc → created_at desc
- [ ] Drag card between columns triggers `PATCH /api/issues/{id}` with status update
- [ ] Optimistic update: card moves immediately, reverts on server rejection (409/422)
- [ ] Sidebar filters (status, priority, category) filter the board in real-time
- [ ] Pagination: infinite scroll or "Load more" within each column
- [ ] Loading state: skeleton cards during initial fetch
- [ ] Empty state per column when no matching issues

### Business Rules
- Drag = status change only (no priority/category change via drag)
- Optimistic locking: send `updated_at` on PATCH; 409 → revert card + toast error
- Filter by status hides/shows entire columns; filter by priority/category filters within visible columns
- needs_attention badge: visual indicator (e.g., orange dot or flame icon)

### Out of Scope
- Issue create modal (03.03)
- Issue detail slide-over (03.04)
- Comment UI (03.05)
- SSE (03.06)
