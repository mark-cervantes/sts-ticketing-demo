# Task 03.03.00: Issue Create Modal

## Business Requirements

### Problem
Users need a way to create new issues from the dashboard without leaving the Kanban view.

### User Story
As a user, I want to click "+ New Issue" and fill a centered modal form to create an issue, so I can quickly add issues without navigating away.

### Acceptance Criteria
- [ ] Centered modal triggered by "+ New Issue" button in sidebar
- [ ] Form fields: title (required), description (required, textarea), priority (select), category (select with inline "add new" option), deadline (optional date picker), visibility (private/public toggle)
- [ ] Category select fetches from `GET /api/categories`; inline add sends `POST /api/categories`
- [ ] Submit sends `POST /api/issues`; on success: close modal, optimistically add card to "Open" column, toast success
- [ ] Validation: inline errors from 422 response mapped to fields
- [ ] Loading state on submit button
- [ ] Escape key or backdrop click closes modal (with unsaved changes confirmation if dirty)
- [ ] URL does NOT change when modal opens (modal is overlay-only)

### Business Rules
- Default priority: medium
- Default visibility: private
- Category list refreshes after inline add
- Description placeholder: "Describe the issue..." (per SPEC §9.7)

### Out of Scope
- Issue editing (03.04 slide-over)
- Drag-drop (03.02)
