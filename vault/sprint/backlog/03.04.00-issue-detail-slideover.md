# Task 03.04.00: Issue Detail Slide-Over Panel

## Business Requirements

### Problem
Users need to view and edit issue details, see the AI summary, manage sharing, and change settings — all without leaving the Kanban dashboard.

### User Story
As a user, I want to click an issue card to open a slide-over panel showing full details, so I can view and edit issues in context.

### Acceptance Criteria
- [ ] Click card → right-side slide-over panel (dashboard stays visible underneath)
- [ ] URL updates to `/dashboard?issue={id}` — back button closes, direct link opens
- [ ] Shows: title (editable inline), description (editable), priority (editable select), status (editable select), category (editable select), deadline (editable date picker), visibility toggle
- [ ] AI summary section: shows summary text + suggested_next_action when status=ready; shows spinner when processing; shows "Summary unavailable" when failed
- [ ] Edit fields trigger `PATCH /api/issues/{id}` with optimistic locking (send `updated_at`)
- [ ] 409 conflict → toast error + reload fresh data from server
- [ ] Delete action via "..." menu → confirm dialog → `DELETE /api/issues/{id}` → close panel, remove card
- [ ] Loading state: skeleton loader while fetching `GET /api/issues/{id}`
- [ ] Close via X button, Escape key, or clicking outside panel

### Business Rules
- Editing description re-triggers AI summary (status goes to "processing" — handled by backend)
- Status change via select in slide-over should also update the Kanban column position
- needs_attention flag displayed prominently if true
- Share section placeholder (wired in 04.01)

### Out of Scope
- Comment thread UI (03.05)
- SSE live updates (03.06)
- Share management UI (04.04)
