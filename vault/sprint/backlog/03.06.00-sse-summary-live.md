# Task 03.06.00: SSE Client for Live Summary Updates

## Business Requirements

### Problem
When an AI summary is processing, the user shouldn't need to refresh to see the result. The summary section should update live.

### User Story
As a user, I want to see the AI summary appear automatically when it's ready, without refreshing the page.

### Acceptance Criteria
- [ ] Backend: `GET /api/issues/{id}/stream` SSE endpoint (auth required)
- [ ] Emits events on `summary_status` transitions (processing → ready, processing → failed)
- [ ] Event payload includes: summary_status, summary text (when ready), suggested_next_action (when ready)
- [ ] Frontend: when issue detail slide-over is open and summary_status is "processing", connect to SSE
- [ ] On "ready" event: update summary section with new text, show success indicator
- [ ] On "failed" event: show "Summary unavailable" message
- [ ] Auto-reconnect on disconnect (EventSource built-in, or manual retry)
- [ ] Close SSE connection when slide-over closes or issue changes
- [ ] Graceful degradation: if SSE connection fails, fall back to polling every 10s

### Business Rules
- SSE stream is per-issue, auth-gated
- Only connect when summary is actually processing (don't open unnecessary connections)
- SummaryCompleted event (02.04) triggers the SSE push

### Out of Scope
- Real-time comment updates
- Presence indicators
- WebSocket upgrade
