# Task 04.02.00: Scheduler for needs_attention Recalculation

## Business Requirements

### Problem
The `needs_attention` flag is computed on issue create/update, but deadlines pass with time. An issue that wasn't urgent yesterday may need attention today if its deadline is approaching. A scheduled job should periodically recalculate.

### User Story
As a user, I want issues approaching their deadline to automatically get flagged as needing attention, even if I haven't edited them recently.

### Acceptance Criteria
- [ ] Artisan command `issues:recalculate-attention` that recalculates `needs_attention` for all open/in-progress issues
- [ ] Laravel scheduler runs the command every 15 minutes
- [ ] Only issues where `needs_attention` actually changed get updated (no unnecessary writes)
- [ ] Command outputs count of issues updated
- [ ] PHPUnit test for the command
- [ ] All tests pass

### Business Rules
- `needs_attention` = true when: priority is critical OR deadline is within 24 hours (SPEC §6.6)
- Resolved issues are excluded from recalculation
- Soft-deleted issues are excluded

### Out of Scope
- Real-time push of attention changes to frontend (would need websocket/SSE extension)
