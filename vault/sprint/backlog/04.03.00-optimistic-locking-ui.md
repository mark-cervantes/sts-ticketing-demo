# Task 04.03.00: Optimistic Locking UI Feedback

## Business Requirements

### Problem
When two users edit the same issue simultaneously, the second save gets a 409 conflict. The current UI shows a toast, but should offer better UX — show what changed and let the user retry.

### User Story
As a user, I want to see a clear conflict message when my edit conflicts with another user's, so I can review the changes and retry.

### Acceptance Criteria
- [ ] On 409 from PATCH, show a conflict dialog (not just toast)
- [ ] Conflict dialog shows: "This issue was updated by another user. Your changes were not saved."
- [ ] "Reload & Retry" button: fetches fresh data, keeps the slide-over open
- [ ] "Discard" button: fetches fresh data, discards user's pending changes
- [ ] After reload, `updated_at` is refreshed so next save works
- [ ] Works for both slide-over edits and drag-drop status changes
- [ ] All tests pass

### Business Rules
- 409 conflict should never silently lose data
- The fresh reload should update the entire issue (another user may have changed multiple fields)

### Out of Scope
- Showing a diff of what the other user changed (too complex for this sprint)
- Real-time conflict prevention (would need websockets)
