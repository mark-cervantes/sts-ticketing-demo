# Task 04.04.00: Share UI in Issue Detail Slide-Over

## Business Requirements

### Problem
Issue owners need a UI to manage sharing directly from the issue detail slide-over.

### User Story
As an issue owner, I want to add, modify, and remove shares from the issue detail panel, so I can control who has access.

### Acceptance Criteria
- [ ] Share section in issue detail slide-over (replaces "Sharing — coming soon" placeholder)
- [ ] Shows list of current shares: user email/name, permission level badge
- [ ] "Add share" input: email field + permission dropdown (view/comment/edit) + add button
- [ ] Permission dropdown to change existing share level (PATCH)
- [ ] "x" button to remove a share (DELETE with confirmation)
- [ ] Only visible to issue owner
- [ ] Inline validation errors (invalid email, user not found, already shared)
- [ ] Toast on success/failure
- [ ] All tests pass

### Business Rules
- Share section only visible when current user is the issue owner
- Permission levels display as human-readable labels: "Can view", "Can comment", "Can edit"
- Adding a share for an already-shared user should update the permission (upsert behavior from 04.01)

### Out of Scope
- Email notifications
- Batch sharing
