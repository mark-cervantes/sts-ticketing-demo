# Task 05.01.00: Production Dockerfile + Docker Compose

## Business Requirements

### Problem
Sail's `compose.yaml` is dev-only — it mounts source code, runs Vite HMR, and lacks production optimizations. Deployment to 192.168.254.140 requires a production-ready Docker setup with pre-built assets, optimized PHP, and proper service orchestration.

### User Story
As a deployer, I want to run `docker compose up -d` on the production host and have the entire app running with migrations, seeds, and all services.

### Acceptance Criteria
- [ ] `Dockerfile` (multi-stage): composer install → npm build → PHP 8.4 FPM production image
- [ ] `docker-compose.prod.yml` with services: app (Laravel + FPM), nginx (or Caddy), postgres, redis, horizon, scheduler
- [ ] `.dockerignore` excludes dev artifacts (node_modules, .git, tests, vendor)
- [ ] Entrypoint script: runs migrations, caches config/routes/views, optionally seeds
- [ ] `APP_ENV=production`, `APP_DEBUG=false` in production compose
- [ ] Horizon runs as a separate service (not in the app container)
- [ ] Scheduler runs `php artisan schedule:work` as a separate service
- [ ] Health check on app service
- [ ] Frontend assets pre-built and served from `public/build/`
- [ ] Total image size < 250MB (no node_modules or dev dependencies in final stage)
- [ ] `docker compose -f docker-compose.prod.yml up -d` starts everything
- [ ] All tests pass in dev (no breakage to existing setup)

### Business Rules
- Dev setup (Sail) must remain untouched — production compose is a separate file
- Postgres data persisted via named volume
- Redis ephemeral (cache/queue — acceptable to lose on restart)
- Queue connection must be redis in production
- Session driver must be redis in production (not file)

### Out of Scope
- Caddy reverse proxy config (05.02)
- SSL/TLS certificates
- CI/CD pipeline
- Kubernetes / cloud orchestration

## Technical Guidance

- **Dockerfile stage 1 (composer):** Use `composer install --no-dev --optimize-autoloader` — `composer.json` has 8 dev-only packages (sail, pint, pail, boost, breeze, pao, faker, mockery) that must not ship.
- **Dockerfile stage 2 (node):** The build script in `package.json` is `vue-tsc && vite build` — run `npm ci && npm run build` then copy only `public/build/` to the final stage; node and `node_modules` must not enter the production image.
- **Dockerfile final stage:** Base on `php:8.4-fpm-alpine`; install extensions `pdo_pgsql`, `pgsql`, `redis`, `pcntl`, `bcmath`, `gd`, `intl`, `zip`, `opcache` — Sail's Dockerfile (`vendor/laravel/sail/runtimes/8.5/Dockerfile`) installs all of these via apt; production needs them via `docker-php-ext-install` / PECL; `pcntl` is required by Horizon for signal handling.
- **`REDIS_CLIENT=phpredis`** is set in `.env.example` line 45 — so `ext-redis` (PECL) is required, not the predis Composer package.
- **Entrypoint must set these env overrides** beyond what the task already lists: `SESSION_DRIVER=redis` (defaults to `database` in `config/session.php:21`), `CACHE_STORE=redis` (defaults to `database` in `config/cache.php:18`), `DB_CONNECTION=pgsql` (defaults to `sqlite` in `config/database.php:20`).
- **Horizon service:** Reuse the same app image with command `php artisan horizon`; `config/horizon.php:216-222` already defines a `production` environment block with `maxProcesses=10` — no config change needed, but the container must set `APP_ENV=production` for that block to activate.
- **Scheduler service:** Command is `php artisan schedule:work` (not `schedule:run`) — it's a long-running process suitable for a container; same app image, no cron needed.
- **Postgres:** Match dev's `postgres:18-alpine` image (`compose.yaml:28`); volume mount to `/var/lib/postgresql/data` (not `/var/lib/postgresql` as Sail uses — Sail's mount path is non-standard and includes the version subfolder).
- **Nginx config:** Since Caddy reverse proxy is out of scope (05.02), the `docker-compose.prod.yml` should include an `nginx:alpine` sidecar serving FPM via fastcgi on port 9000; the nginx config must set `root /var/www/html/public` and pass `.php` to the `app` service — Caddy (05.02) will then proxy to this nginx.
- **`.dockerignore`:** Must exclude `.git/`, `node_modules/`, `vendor/`, `tests/`, `storage/logs/`, `.env`, `docker-compose.yaml` (Sail), `.sail/` — but must NOT exclude `public/` or `resources/` (needed at build time).
