# Task 05.03.00: README + Architecture Documentation

## Business Requirements

### Problem
The project needs comprehensive documentation for submission/evaluation — explaining the architecture, how to run it, what was built, and design decisions made.

### User Story
As an evaluator, I want to read a clear README that explains what this project is, how to run it, and what architectural decisions were made, so I can assess the work.

### Acceptance Criteria
- [ ] `README.md` at project root with:
  - Project overview (what it does, key features)
  - Tech stack summary
  - "Try It Locally" section (prerequisites, one-command setup, seeded credentials)
  - Architecture overview with Mermaid diagram (request flow, service layer, AI pipeline)
  - API endpoint summary table
  - Testing section (how to run, what's covered, test counts)
  - Design decisions / ADR summary (link to vault/docs/adr/)
  - Screenshots of the Kanban dashboard (light + dark mode)
- [ ] Architecture diagram shows: Browser → Inertia → Controller → Service → Model, Queue → GenerateSummaryJob → SummaryManager → Drivers, SSE stream
- [ ] All existing ADRs referenced and summarized
- [ ] Screenshots captured and committed to `docs/screenshots/`
- [ ] All tests pass (no breakage)

### Business Rules
- README must be self-contained — evaluator should not need to dig into vault/ to understand the project
- Screenshots should show real seeded data, not empty states
- Mermaid diagrams render in GitHub markdown

### Out of Scope
- API documentation (Swagger/OpenAPI)
- Inline code documentation (PHPDoc is already in place)
