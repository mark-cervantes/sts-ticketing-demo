# Task 09.02: AI Chat — Frontend (Chat Panel in Issue Detail Sheet)

## Business Requirements

### Problem
The backend API for AI chat exists (task 09.01), but there's no UI for users to interact with it. Users need a chat interface embedded in the issue detail sheet where they can send messages and see streamed AI responses in real time.

### User Story
As a user viewing an issue, I want a chat panel where I can type questions about the issue and see AI responses stream in real time, so I can quickly understand and investigate the issue.

### UX Design

#### Placement
The chat panel lives inside the `IssueDetailSheet` as a collapsible section or tab. Two approaches (tech-lead decides):
- **Option A: Tab** — Add a "Chat" tab alongside the existing content, so the sheet toggles between "Details" and "AI Chat"
- **Option B: Collapsible section** — Below the summary section, a "Chat with AI" expandable area

#### Chat Interface
```
┌─────────────────────────────────────┐
│  🤖 AI Chat                    ▾   │
├─────────────────────────────────────┤
│                                     │
│  You: What timeout value did        │
│  Carol mention?                     │
│                                     │
│  AI: Carol Chen confirmed the       │
│  gateway hard-times out at 30       │
│  seconds during 09:00–11:00 UTC.    │
│  The gateway team suggested         │
│  increasing the client timeout      │
│  to 60 seconds as a short-term      │
│  fix.                               │
│                                     │
│  You: Is there a retry mechanism?   │
│                                     │
│  AI: Based on the conversation,     │
│  Alice Johnson recommended...█      │  ← streaming cursor
│                                     │
├─────────────────────────────────────┤
│  [Type a message...        ] [Send] │
└─────────────────────────────────────┘
```

#### Streaming UX
- AI responses appear token-by-token as they stream from the SSE endpoint
- While streaming: show a pulsing cursor/indicator, disable the send button
- After `[DONE]`: cursor disappears, send button re-enables
- On error: show an inline error message in the chat area, not a toast

#### Message Rendering
- User messages: right-aligned, primary background (like a chat bubble)
- AI messages: left-aligned, muted background
- AI messages render markdown (bold, italic, bullet points, code) using a lightweight renderer
- Timestamps shown on hover or as subtle text below each message

### Acceptance Criteria
- [ ] Chat panel renders inside IssueDetailSheet (tab or collapsible — tech-lead decides)
- [ ] `useIssueChat(issueId)` composable handles: loading history, sending messages, SSE streaming
- [ ] Chat history loads on panel open (GET `/api/issues/{issue}/chat`)
- [ ] User can type and send a message (POST `/api/issues/{issue}/chat`)
- [ ] AI response streams in real-time — tokens appear as they arrive
- [ ] Streaming indicator (pulsing dot or cursor) visible during AI response
- [ ] Send button disabled while AI is responding
- [ ] Error states: no AI configured (503), rate limited, network error — all shown inline
- [ ] Chat auto-scrolls to bottom on new messages
- [ ] Empty state: "Ask AI anything about this issue" prompt
- [ ] Dark mode support
- [ ] Responsive: works in the sheet's constrained width
- [ ] `make verify-visual` passes with chat panel visible
- [ ] vue-tsc clean

### Business Rules
- Chat panel only shows if the user has view permission on the issue (`can.view`)
- Messages are persistent — reopening the sheet shows previous chat history
- The composable uses `EventSource` or `fetch` with `ReadableStream` for SSE consumption
- Previous chat history is loaded oldest-first, auto-scrolled to bottom
- Rate limit feedback: if 429 returned, show "You've reached the message limit. Try again in X minutes."

### Out of Scope
- Chat message editing/deletion
- File attachments
- @mentions in chat
- Chat export
- Typing indicators for other users
