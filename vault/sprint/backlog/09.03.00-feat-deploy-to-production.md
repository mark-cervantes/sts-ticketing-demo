# Task 09.03: Deploy Latest Dev to Production

## Business Requirements

### Problem
Multiple sprints of work (AI prompt improvements, category management, auth page polish, creator display, skimmable summaries, DB-stored prompts, auto-regeneration on comments) are on `dev` but not yet deployed to production at `https://sts-demo.betamaxgroup.tech`.

### Steps
1. Ensure all tests pass on dev: `./vendor/bin/sail test --compact`
2. Ensure vue-tsc clean: `./vendor/bin/sail npx vue-tsc --noEmit`
3. Merge dev → main: `git checkout main && git merge --no-ff dev -m "release: Sprint 7 + AI improvements"`
4. Push main: `git push origin main`
5. CI workflow triggers on push to main → self-hosted runner deploys
6. Verify production: `curl -s -o /dev/null -w "%{http_code}" https://sts-demo.betamaxgroup.tech`
7. Run `summaries:retry-stuck` on production if needed (new prompt format)

### Acceptance Criteria
- [ ] `main` is up to date with `dev`
- [ ] CI pipeline passes (GitHub Actions)
- [ ] Production returns 200
- [ ] New features visible on production (category settings, "You" badge, skimmable summaries)

### Out of Scope
- Infrastructure changes
- New environment variables (all AI settings are already DB-stored)
