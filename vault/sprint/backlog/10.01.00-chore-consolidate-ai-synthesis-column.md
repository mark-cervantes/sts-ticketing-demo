# Task 10.01: Consolidate AI Output Columns into `ai_synthesis` JSONB

## Context
Per ADR-004, AI-generated fields should live in a single JSON column, not spread across individual columns. Currently `issues` has `summary`, `suggested_next_action`, `suggested_next_ticket` as separate TEXT columns. This task consolidates them.

## Steps
1. Add `ai_synthesis JSONB DEFAULT '{}'` column to `issues`
2. Data migration: for each issue, combine existing column values into the JSON:
   ```json
   { "summary": "...", "suggested_next_ticket": "..." }
   ```
3. Update `Issue` model: replace individual fillable fields with `ai_synthesis` cast as `array` or a custom DTO
4. Update `LlmDriver`: store parsed JSON directly into `ai_synthesis` instead of mapping to individual fields
5. Update `RulesDriver`: same
6. Update `GenerateSummaryJob`: write to `ai_synthesis` column
7. Update `IssueResource`: read from `ai_synthesis`
8. Update `IssueSseController`: read from `ai_synthesis`
9. Update `SummaryResult` value object: simplify to just carry the raw parsed array
10. Update frontend: `issue.summary` → `issue.ai_synthesis?.summary`, etc.
11. Drop old columns: `summary`, `suggested_next_action`, `suggested_next_ticket`, `summary_status`
12. Update all tests

## Acceptance Criteria
- [ ] Single `ai_synthesis` JSONB column replaces 3 TEXT columns
- [ ] Zero references to `suggested_next_action` in codebase (completes 09.05)
- [ ] Adding a new AI output key requires ONLY: prompt change + frontend change
- [ ] All tests pass
- [ ] Existing summaries preserved via data migration
