## What To Build

`components.json` (shadcn-vue CLI metadata) contains `tailwind.config: "tailwind.config.js"`, but this project uses Tailwind v4 which is config-less (theming via `@theme inline` in `resources/css/app.css`). The phantom path doesn't break anything at runtime — it's only consulted by `shadcn-vue` CLI when adding new components.

**Investigate first** — current shadcn-vue CLI may have updated handling for Tailwind v4. If a clean v4 mode is supported, switch to it. Otherwise, set the field to empty string `""` or remove if the CLI tolerates absence.

Verification: after the change, `./vendor/bin/sail npx shadcn-vue@latest add tooltip` (or any non-installed component) must still succeed without errors.
