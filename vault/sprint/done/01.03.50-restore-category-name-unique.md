## Background

SPEC §4.3 and ADR-006 specify `categories.name` is **unique** at the DB level. During 01.03.00 implementation, the constraint was dropped because the QA test `test_slug_collision_handled` creates two `Category` rows with identical names ("Bug Reports") to verify slug suffixing. With DB-unique on `name`, the test would fail on the second insert before slug logic runs.

## What To Build

1. **Restore unique constraint** via new migration:
   - `./vendor/bin/sail artisan make:migration add_unique_to_categories_name`
   - Add `$table->unique('name')` on up, drop on down.

2. **Rewrite QA collision test** to use near-identical names that produce the same slug through `Str::slug()` normalization:
   - `tests/Unit/Models/CategoryTest.php::test_slug_collision_handled`
   - Use distinct names: `'Bug Reports'` + `'Bug  Reports'` (extra space) + `'BUG REPORTS'` — all slugify to `'bug-reports'`.
   - Assert: first slug = `bug-reports`, second = `bug-reports-2`, third = `bug-reports-3`.

3. **Verify:**
   - `./vendor/bin/sail artisan migrate` (forward only — don't `migrate:fresh` to preserve any seeded data)
   - `./vendor/bin/sail test --filter='CategoryTest'` — all green
   - `./vendor/bin/sail test --compact` — full suite green
