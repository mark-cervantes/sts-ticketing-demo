# Task 03.03.00: Issue Create Modal

## Business Requirements

### Problem
Users need a way to create new issues from the dashboard without leaving the Kanban view.

### User Story
As a user, I want to click "+ New Issue" and fill a centered modal form to create an issue, so I can quickly add issues without navigating away.

### Acceptance Criteria
- [x] Centered modal triggered by "+ New Issue" button in sidebar
- [x] Form fields: title (required), description (required, textarea), priority (select), category (select with inline "add new" option), deadline (optional date picker), visibility (private/public toggle)
- [x] Category select fetches from `GET /api/categories`; inline add sends `POST /api/categories`
- [x] Submit sends `POST /api/issues`; on success: close modal, optimistically add card to "Open" column, toast success
- [x] Validation: inline errors from 422 response mapped to fields
- [x] Loading state on submit button
- [x] Escape key or backdrop click closes modal (with unsaved changes confirmation if dirty)
- [x] URL does NOT change when modal opens (modal is overlay-only)

### Business Rules
- Default priority: medium
- Default visibility: private
- Category list refreshes after inline add
- Description placeholder: "Describe the issue..." (per SPEC §9.7)

### Out of Scope
- Issue editing (03.04 slide-over)
- Drag-drop (03.02)

## Technical Guidance

### Architecture Notes

- Use `Dialog`/`DialogContent`/`DialogHeader`/`DialogFooter` from `@/components/ui/dialog` (reka-ui based) — **not** Sheet, which is reserved for slide-over panels (Sheet is already used for mobile sidebar in `AppLayout.vue:39`)
- Create composable `resources/js/composables/useCreateIssue.ts` — owns reactive form state, `submit()`, `resetForm()`, field error refs, `isSubmitting` flag, and `isDirty` computed; follows existing composable pattern in `composables/useDarkMode.ts`
- API calls via `window.axios` (configured in `bootstrap.ts:3`) — Axios auto-reads `XSRF-TOKEN` cookie for same-origin POST requests, so no manual CSRF header needed
- Category select needs a second composable or inline logic: `GET /api/categories` returns `[{ id, name, slug }]` (see `CategoryController::index`); inline add via `POST /api/categories` returns `{ id, name, slug }` with 201 — append to local list and auto-select the new entry
- Map 422 validation errors from `response.data.errors` (Laravel's `{ errors: { field: [messages] } }` shape per `StoreIssueRequest`) to per-field error refs
- Priority select options must match `Priority` enum values exactly: `low`, `medium`, `high`, `critical` (`app/Enums/Priority.php`)
- Visibility toggle between `private` / `public` must match `Visibility` enum values (`app/Enums/Visibility.php`); field is `sometimes|nullable` in `StoreIssueRequest:38` — send only when user changes from default
- Success toast via `toast()` from `vue-sonner` (already wired: `Toaster` is in `AppLayout.vue:52`)

### Missing shadcn-vue Components — Install Before Implementing

- **Textarea** — `npx shadcn-vue@latest add textarea` (description field)
- **Label** — `npx shadcn-vue@latest add label` (form field labels)
- **Switch** — `npx shadcn-vue@latest add switch` (visibility toggle, or use Select with two options)
- **Calendar + Popover** — `npx shadcn-vue@latest add calendar popover` (deadline date picker); alternatively use a native `<input type="date">` if avoiding added complexity

### Technical DoD

**Visual:**
- Modal centered, backdrop dismissible, responsive (full-width on mobile)
- All form fields render with labels, correct input types, and error message slots
- Loading spinner/disabled state on submit button during API call
- Toast appears on success; modal closes and sidebar/board state is consistent

**Code:**
- No business logic in Vue components — all API/state logic in `useCreateIssue` composable
- Form defaults: `priority: 'medium'`, `visibility: 'private'` (matching business rules)
- `isDirty` computed tracks whether any field differs from defaults — guards backdrop/Escape close with confirmation
- Category list re-fetched after inline add (not full page reload)
- 422 error response mapped to field-level refs, cleared on next input

### Gotchas

- **CSRF**: Axios reads `XSRF-TOKEN` cookie automatically for same-origin requests — do NOT manually set `X-CSRF-TOKEN` header or read `<meta>` tags; ensure `withCredentials` is not explicitly set to `false`
- **Dialog vs Sheet**: `Dialog` = centered modal (this task); `Sheet` = slide-over panel (reserved for 03.04 issue edit and mobile sidebar); mixing them violates UX consistency
- **Category refresh after add**: After `POST /api/categories` succeeds, push the returned `{ id, name, slug }` into the local reactive categories array and set `category_id` to the new `id` — do NOT re-fetch the full list (avoids flicker and race)
- **Dirty state + Dialog close**: reka-ui `Dialog` fires `update:open` on Escape/backdrop; intercept via `@update:open` to show confirmation if `isDirty.value === true` before allowing close
- **deadline_at validation**: Backend rule is `after:now` (`StoreIssueRequest:39`) — front-end date picker must disable past dates to avoid unnecessary 422 round-trips
- **Optimistic add to Kanban**: The Kanban board (03.02) may not exist yet — emit a `issue-created` event or use a shared store; if 03.02 is not merged, the toast alone is sufficient for this task
