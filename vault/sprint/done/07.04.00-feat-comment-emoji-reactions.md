# Task 07.04: Add Emoji Reactions to Comments

## Business Requirements

### Problem
Users have no lightweight way to acknowledge or respond to comments without writing a full reply. Emoji reactions (like GitHub's 👍 👎 😄 🎉 😕 ❤️) provide quick feedback without cluttering the comment thread.

### User Story
As a user, I want to react to comments with emojis so that I can quickly acknowledge or respond without writing a full comment.

### Acceptance Criteria
- [ ] Each comment shows a reaction button (e.g., smiley face icon)
- [ ] Clicking the reaction button opens a picker with a curated set of emojis (👍 👎 😄 🎉 😕 ❤️ 🚀 👀)
- [ ] Clicking an emoji toggles the current user's reaction (add/remove)
- [ ] Reaction counts are shown below each comment (grouped by emoji, e.g., "👍 3 ❤️ 1")
- [ ] Clicking a displayed reaction toggles the current user's reaction on it
- [ ] Users who reacted are shown on hover (tooltip with names)
- [ ] Reactions persist across page reloads
- [ ] Only authenticated users with at least `view` access can react
- [ ] Migration + model + API + UI all included
- [ ] PHPUnit tests for reaction CRUD
- [ ] Visual verification via `make verify-visual`

### Business Rules
- Each user can react with each emoji type only once per comment (toggle on/off)
- Reactions are anonymous in count display but names shown on hover
- Deleting a comment deletes all its reactions (cascade)
- Any user who can view the issue can react to comments on it (lower bar than commenting)

### Out of Scope
- Custom emoji
- Reactions on issues themselves (only on comments)
- Real-time reaction updates via SSE
