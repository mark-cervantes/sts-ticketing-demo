# Task 07.05: Custom Login & Register Pages

## Business Requirements

### Problem
The current login and register pages (`resources/js/Pages/Auth/Login.vue`, `Register.vue`) are the default Laravel Breeze scaffold — generic and unstyled beyond basic Tailwind. They don't reflect the STS product identity or provide a polished first impression.

### Goal
Replace the generic Breeze auth pages with custom-designed, project-branded login and register pages that feel like a real product, not a boilerplate.

### Acceptance Criteria
- [ ] Login page has a distinctive layout (e.g., split-screen, centered card with background, or sidebar illustration)
- [ ] Includes STS branding: app name, tagline, and visual identity consistent with the Kanban dashboard
- [ ] Dark mode support matching the existing design system
- [ ] Register page follows the same visual language as the login page
- [ ] "Remember me" checkbox, forgot password link, and registration link on login
- [ ] Form validation messages display inline (Inertia form errors)
- [ ] Responsive: works well on mobile, tablet, and desktop
- [ ] All existing auth functionality preserved (Breeze backend unchanged)
- [ ] `make verify-visual` passes with screenshots of login and register pages
- [ ] PHPUnit auth tests still pass

### Business Rules
- Backend auth logic (controllers, routes, middleware) must NOT change — only the Vue pages
- Use existing shadcn-vue components (Card, Input, Button, Label, etc.)
- Follow the existing design system tokens (colors, spacing, typography)

### Out of Scope
- OAuth/social login
- Two-factor authentication
- Password reset page redesign (can be a follow-up)

## Technical Guidance

### Grounding
- Read: `resources/js/Pages/Auth/Login.vue` — already uses `GuestLayout`, shadcn Card/Input/Button/Label, `useForm()`, "Sign in" wording
- Read: `resources/js/Pages/Auth/Register.vue` — mirrors Login; same layout contract
- Read: `resources/js/Layouts/GuestLayout.vue` — split-screen layout **already exists**: left `bg-primary text-primary-foreground` brand panel (hidden on mobile with `lg:hidden` flip), right `bg-background` form column; uses `/images/logo-icon.svg` via `brightness-0 invert`
- Read: `resources/css/app.css` — `--primary: oklch(0.55 0.22 250)` light mode; `--primary: oklch(0.65 0.22 250)` dark mode; geometric decorators are `bg-white/5` (opacity-on-brand-color, not hardcoded indigo)
- Confirmed: `public/images/logo.svg` and `public/images/logo-icon.svg` exist
- Confirmed: `resources/js/components/ui/` has card, button, input, label, checkbox, separator, badge, avatar, skeleton — all available
- Confirmed: `app/Http/Controllers/Auth/` — 9 Breeze controllers (off-limits)
- Confirmed: `tests/Feature/Auth/` — 6 PHPUnit test classes; all must continue to pass

### Architecture Notes
- **The split-screen layout already exists in `GuestLayout.vue`** — do not create a new `AuthLayout.vue`. The task is to enrich the content within the two panels, not restructure the layout contract. Both `Login.vue` and `Register.vue` already declare `<GuestLayout>` as their root.
- **Zero backend changes permitted.** `app/Http/Controllers/Auth/AuthenticatedSessionController.php` and `app/Http/Controllers/Auth/RegisteredUserController.php` render Inertia pages by name (`Auth/Login`, `Auth/Register`) and pass `canResetPassword` / `status` props — the Vue component signatures (`defineProps`) must stay compatible.
- **Form handling is `useForm()` from `@inertiajs/vue3`** — already in place in both files. Do not replace with a custom `ref()` + `axios` pattern; Inertia's `form.processing`, `form.errors`, and `form.reset()` callbacks are already wired and tested by `tests/Feature/Auth/AuthenticationTest.php`.
- **Dark mode on the brand panel needs explicit attention.** In `GuestLayout.vue` the panel uses `bg-primary` whose dark-mode value (`oklch(0.65 0.22 250)`) is a lighter indigo. The decorative `bg-white/5` blobs look fine on dark-indigo but wash out on the lighter dark-mode variant — use `bg-primary-foreground/5` or `bg-black/10` for the decorators so they remain visible in both modes, referencing token names from `resources/css/app.css` lines 21–64.
- **Logo usage**: the icon variant (`/images/logo-icon.svg`) is used in the panel with `brightness-0 invert` to force white. The full-wordmark (`/images/logo.svg`) is available if a larger branding moment is desired in the right-hand form panel. Do not embed SVG markup inline — use `<img>` pointing at `public/images/` to avoid Vite asset pipeline involvement.
- **Wording convention**: `<Head title="Sign in" />` is already set in `Login.vue`; the submit button already reads "Sign in" / "Signing in…". Register's title should read "Create account" to match the `CardTitle` already present. Do not introduce "Log in" anywhere — per project convention "Sign in" is the canonical verb.
- **`Checkbox.vue`** lives at `resources/js/Components/Checkbox.vue` (the legacy Breeze component, not a shadcn component). `Login.vue` already imports it as `@/Components/Checkbox` — do not swap it for a shadcn `Switch` or a bare `<input type="checkbox">`.
- **Responsive contract**: `GuestLayout.vue` collapses the brand panel at `lg` breakpoint (`hidden lg:flex`); a mobile logo strip (`lg:hidden`) is shown instead. Any brand panel content changes must remain inside the `<aside>` so the mobile strip is unaffected.
- **`make verify-visual` is a hard gate** — Playwright smoke checks the `/login` and `/register` routes. If new page elements change element roles or landmark structure (e.g., adding a second `<main>`) the smoke spec at `tests/Playwright/smoke.spec.ts` must be extended in the same branch.

### Gotchas
- **Do not create `AuthLayout.vue`** — a second layout would require updating both page files and introduces an orphan layout that future auth pages (confirm-password, verify-email) won't pick up automatically. Extend `GuestLayout.vue` instead.
- **`defineProps` contract is Inertia-enforced**: `Login.vue`'s `canResetPassword?: boolean` and `status?: string` are injected by `AuthenticatedSessionController@create` — if these props are removed or renamed the "Forgot password?" link silently disappears and status flash after password reset breaks, without a PHP error.
- **`brightness-0 invert` on the SVG logo** only works correctly when the SVG is a single-color mark. `/images/logo-icon.svg` is confirmed to be the icon-only file; `/images/logo.svg` is the full wordmark — test rendering of both before deciding which to place in the panel versus the mobile strip.
- **Tailwind v4 `@theme inline` is in use** (`resources/css/app.css` line 21). All brand tokens must be referenced via the semantic utility classes (`bg-primary`, `text-primary-foreground`) — never hardcode `#6366F1` or raw `oklch()` values in Vue templates, as dark-mode overrides won't apply.
