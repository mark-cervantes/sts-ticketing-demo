# Task 09.05: Remove suggested_next_action from Backend Pipeline

## Context
The "Suggested Next Action" banner was removed from the UI (IssueDetailSheet) because it's redundant with the follow-up ticket card and the new AI chat. The backend still generates and stores the field. Clean it up fully.

## What to remove
1. **Migration**: Drop `suggested_next_action` column from `issues` table
2. **Model**: Remove from `Issue::$fillable`
3. **Prompt**: Remove the `suggested_next_action` key from `config/prompts/summary.php` system prompt (change from 3 keys to 2)
4. **LlmDriver**: Remove parsing/validation of `suggested_next_action` from JSON response
5. **SummaryResult**: Remove `suggestedNextAction` property
6. **GenerateSummaryJob**: Remove assignment of `suggested_next_action`
7. **IssueResource**: Remove `suggested_next_action` from API response
8. **IssueSseController**: Remove from SSE payload
9. **AiSettingsController**: Remove from test connection result display
10. **IssueController**: Remove null assignment on issue creation
11. **RulesDriver**: Remove if it generates a suggested_next_action
12. **Tests**: Update any tests asserting on `suggested_next_action`

## Acceptance Criteria
- [ ] Column dropped from DB
- [ ] Zero references to `suggested_next_action` or `suggestedNextAction` in codebase
- [ ] LLM prompt only asks for `summary` + `suggested_next_ticket`
- [ ] All tests pass
- [ ] Regenerate summaries after migration (only 2-key JSON now)
