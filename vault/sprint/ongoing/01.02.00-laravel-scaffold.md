### 2. Vite Config for Sail HMR

Breeze generates `vite.config.ts`. It may not include the Sail-specific HMR settings. The container listens on port `5175` (`VITE_PORT` in `.env`), and the compose service maps `5175:5175`.

**Required final state of `vite.config.ts` `server` block:**

```ts
server: {
    host: '0.0.0.0',               // bind inside container to all interfaces
    hmr: {
        host: 'localhost',          // browser reaches HMR via localhost (host machine)
    },
    watch: {
        usePolling: true,           // needed for Docker volume mounts
    },
},
```

- `server.host` **must** be `0.0.0.0` ŌĆö without it, Vite binds to `127.0.0.1` inside the container and the port-forward from Docker never reaches it.
- `hmr.host` must be `localhost` ŌĆö the browser on the host connects to HMR through the forwarded port.
- `usePolling: true` ensures file-change detection works over Docker bind mounts (Linux host ŌåÆ container can skip this, but it's harmless and safer to include).
- Port is already `5175` via `VITE_PORT` env ŌĆö Breeze/Sail read it automatically. Do **not** hardcode `port: 5175` unless Breeze's config omits it.

**If Breeze's generated config already has `server.host: '0.0.0.0'` and `hmr.host: 'localhost'`, no edit needed.** If it doesn't, patch only the `server` block. Do not remove `watch.ignored` if Breeze sets one.

**Verify ┬¦2:**
```bash

# ./vendor/bin/sail npm run dev
# Then:
curl -s http://localhost:5175/@vite/client | head -1   # should return JS content
```

**Key notes on init:**

- shadcn-vue writes a `components.json` at project root. Inspect it after init to verify `aliases.components` = `@/components` and `aliases.utils` = `@/lib/utils`. These must match the `@` alias in `vite.config.ts` ŌåÆ `resolve.alias` (Breeze sets `@` ŌåÆ `resources/js` by default).
- shadcn-vue init **will modify `resources/css/app.css`** ŌĆö it injects its own theme tokens. We will overwrite the theming section in ┬¦4, so order matters: run init first, then apply our token set.

### 4. Tailwind v4 Theming ŌĆö `resources/css/app.css`

After shadcn-vue init, `resources/css/app.css` will have shadcn's default tokens. Replace/merge the theming to match SPEC ┬¦2 (primary color changeable in ONE place). Follow `tailwind-theme.reference[ux-architect]` for structure.

**Required structure (full file ŌĆö order matters per Tailwind v4 spec):**

```css
/* ŌæĀ Must be first line ŌĆö loads Tailwind engine */
@import 'tailwindcss';

/* ŌæĪ Source paths (Breeze + shadcn components) */
@source '../views/**/*.blade.php';
@source '../js/**/*.vue';
@source '../js/**/*.ts';

/* Ōæó Dark mode variant ŌĆö class-based on <html> */
@custom-variant dark (&:is(.dark *));

/* ŌæŻ Map semantic vars ŌåÆ Tailwind utility namespaces */
/* MUST use @theme inline ŌĆö plain @theme bakes values at build time, dark overrides won't cascade */
@theme inline {
  --color-background:             var(--background);
  --color-foreground:             var(--foreground);
  --color-card:                   var(--card);
  --color-card-foreground:        var(--card-foreground);
  --color-popover:                var(--popover);
  --color-popover-foreground:     var(--popover-foreground);
  --color-primary:                var(--primary);
  --color-primary-foreground:     var(--primary-foreground);
  --color-secondary:              var(--secondary);
  --color-secondary-foreground:   var(--secondary-foreground);
  --color-muted:                  var(--muted);
  --color-muted-foreground:       var(--muted-foreground);
  --color-accent:                 var(--accent);
  --color-accent-foreground:      var(--accent-foreground);
  --color-destructive:            var(--destructive);
  --color-destructive-foreground: var(--destructive-foreground);
  --color-border:                 var(--border);
  --color-input:                  var(--input);
  --color-ring:                   var(--ring);

  --radius-sm: calc(var(--radius) * 0.6);
  --radius-md: calc(var(--radius) * 0.8);
  --radius-lg: var(--radius);
  --radius-xl: calc(var(--radius) * 1.4);

  /* Keep any --font-sans that Breeze/shadcn set */
}

/* Ōæż Light mode token values ŌĆö THE ONLY PLACE actual color values live */
/* To change primary color: edit --primary here. One place. Done. */
:root {
  --radius: 0.5rem;
  --background:             oklch(1 0 0);
  --foreground:             oklch(0.145 0 0);
  --card:                   oklch(1 0 0);
  --card-foreground:        oklch(0.145 0 0);
  --popover:                oklch(1 0 0);
  --popover-foreground:     oklch(0.145 0 0);
  --primary:                oklch(0.55 0.22 250);       /* ŌåÉ change this ONE value for brand color */
  --primary-foreground:     oklch(0.985 0 0);
  --secondary:              oklch(0.96 0.005 250);
  --secondary-foreground:   oklch(0.205 0.015 250);
  --muted:                  oklch(0.96 0.005 250);
  --muted-foreground:       oklch(0.556 0.015 250);
  --accent:                 oklch(0.96 0.005 250);
  --accent-foreground:      oklch(0.205 0.015 250);
  --destructive:            oklch(0.577 0.245 27.33);
  --destructive-foreground: oklch(0.985 0 0);
  --border:                 oklch(0.922 0 0);
  --input:                  oklch(0.922 0 0);
  --ring:                   oklch(0.55 0.22 250);       /* matches --primary */
}

/* Ōæź Dark mode ŌĆö same tokens, dark values */
.dark {
  --background:             oklch(0.145 0 0);
  --foreground:             oklch(0.985 0 0);
  --card:                   oklch(0.205 0 0);
  --card-foreground:        oklch(0.985 0 0);
  --popover:                oklch(0.205 0 0);
  --popover-foreground:     oklch(0.985 0 0);
  --primary:                oklch(0.65 0.22 250);       /* slightly lighter primary for dark bg */
  --primary-foreground:     oklch(0.145 0 0);
  --secondary:              oklch(0.269 0.015 250);
  --secondary-foreground:   oklch(0.985 0 0);
  --muted:                  oklch(0.269 0.015 250);
  --muted-foreground:       oklch(0.65 0.015 250);
  --accent:                 oklch(0.269 0.015 250);
  --accent-foreground:      oklch(0.985 0 0);
  --destructive:            oklch(0.577 0.245 27.33);
  --destructive-foreground: oklch(0.985 0 0);
  --border:                 oklch(1 0 0 / 10%);
  --input:                  oklch(1 0 0 / 15%);
  --ring:                   oklch(0.65 0.22 250);
}

/* Ōæ” Base layer defaults */
@layer base {
  * { @apply border-border; }
  body { @apply bg-background text-foreground; }
}
```

**Key points for coder:**
- If shadcn-vue init produced its own `:root`/`.dark` blocks with different token names or HSL values, **replace them** with the OKLCH set above. Token *names* must match what shadcn-vue components reference (they use `bg-primary`, `text-muted-foreground`, etc. which resolve via `@theme inline` mappings).
- The hue `250` in the primary OKLCH is a placeholder (blue-ish). It can be changed ŌĆö that's the "one place" requirement from SPEC ┬¦2.
- `@custom-variant dark (&:is(.dark *))` is mandatory for Tailwind v4 class-based dark mode. Without it, `dark:` utilities compile but never activate.
- `@import 'tailwindcss'` MUST be line 1. CSS spec ignores `@import` placed after any non-import rule.
- Keep `@source` paths covering both Blade templates and Vue components so Tailwind scans everything.

### 8. Pitfalls & Warnings

1. **Vite port binding:** Container AND host both use `5175`. The `server.host` in `vite.config.ts` MUST be `'0.0.0.0'` for Sail containers. If missing, Vite binds to `127.0.0.1` inside the container ŌåÆ HMR unreachable from host browser.

2. **Boost MCP bootability:** Boost MCP invokes `php artisan boost:mcp` ŌĆö if the app fails to boot (e.g., missing migration, broken config), MCP tools in this project will fail. Commit `wip:` after each section so the app stays bootable.

3. **Existing tests are sacred:** `tests/Feature/ExampleTest.php` and `tests/Unit/ExampleTest.php` must NOT be modified or deleted. Breeze adds new test files alongside them. If Breeze overwrites `ExampleTest`, restore it from git: `git checkout HEAD -- tests/Feature/ExampleTest.php tests/Unit/ExampleTest.php`.

4. **Breeze overwrites `resources/css/app.css`:** The original Tailwind v4 `@theme` block with `--font-sans` will be replaced. That's expected ŌĆö shadcn-vue init and our ┬¦4 theming replace it entirely. Order is: Breeze ŌåÆ shadcn-vue init ŌåÆ our theming (┬¦4).

5. **Breeze rewrites `vite.config.js` ŌåÆ `vite.config.ts`:** The original `vite.config.js` (with `bunny()` font plugin and `@tailwindcss/vite`) gets replaced. Breeze's generated `vite.config.ts` uses `laravel-vite-plugin` and `@vitejs/plugin-vue`. The `@tailwindcss/vite` plugin should still be present ŌĆö verify after install.

6. **shadcn-vue `components.json` alias:** The `--template laravel` flag should set `aliases.components` ŌåÆ `@/components` and `aliases.utils` ŌåÆ `@/lib/utils`. Verify this matches the `@` alias in `vite.config.ts` ŌåÆ `resolve.alias` (Breeze sets `@` ŌåÆ `./resources/js`). Mismatch = import errors at build time.

7. **Horizon REQUIRES Redis driver:** `QUEUE_CONNECTION=database` (current `.env`) makes Horizon non-functional. The `.env` must be updated to `QUEUE_CONNECTION=redis` BEFORE testing Horizon. The Redis container is already in the Sail stack.

8. **Sail user/group:** `WWWUSER=1000`, `WWWGROUP=1000` in `.env`. File ownership should be fine since the host UID typically matches. If `npm install` or `artisan` commands produce permission errors, run `./vendor/bin/sail root-shell` and `chown -R sail:sail /var/www/html/node_modules`.

9. **HMR verification is a one-shot check:** The coder verifies HMR by running `sail npm run dev` in a separate terminal, confirming the Vite client is accessible via curl, then stopping it. Do NOT leave a long-running `npm run dev` process from within the agent.

10. **Pint after PHP changes:** After Breeze and Horizon install, run `./vendor/bin/sail pint --dirty --format agent` to fix formatting on any modified PHP files per Boost guidelines.
