## What To Build

Create model factories and database seeders that produce realistic demo data.

### Factories (database/factories/)
- **UserFactory** — realistic names/emails via Faker
- **CategoryFactory** — realistic category names
- **IssueFactory** — configurable states for each status, priority, visibility. Respects enum values. Can generate with/without deadline.
- **CommentFactory** — realistic body text
- **IssueShareFactory** — configurable permission level

### Seeders (database/seeders/)
- **CategorySeeder** — creates default categories: billing, technical, account, general, bug, feature-request
- **UserSeeder** — 5 users with distinct realistic names/emails
- **IssueSeeder** — 15-20 issues spanning:
  - All priority levels
  - All status values
  - Mix of private/public visibility
  - Some with deadlines, some without
  - At least 2 with summary_status=ready (pre-filled summary)
  - At least 3 with needs_attention=true
- **CommentSeeder** — 2-4 comments per issue (30+ total)
- **IssueShareSeeder** — 2-3 issues shared between users
- **DatabaseSeeder** — orchestrates all above in correct order

### Requirements per SRS §6
- Minimum: 3+ users, 5+ issues, comments across issues
- At least 1 issue with summary_status=ready
- At least 1 with needs_attention=true
- All seeded categories populated

## Tests Required
- Feature: `php artisan migrate:fresh --seed` runs without error
- Verify seeded data: correct counts, relationships intact
