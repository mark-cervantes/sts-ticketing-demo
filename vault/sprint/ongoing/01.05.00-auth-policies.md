## What To Build

Customize Breeze auth and implement Laravel Policies for the ownership + sharing authorization model.

### Auth Customization
- Breeze already provides login/register — verify it works with our User model
- Ensure authenticated user data is shared via Inertia (HandleInertiaRequests middleware)
- No roles — just authenticated vs. not

### Policies (app/Policies/)

**IssuePolicy:**
- `viewAny()`: any authenticated user (sees own + shared + public in list)
- `view($user, $issue)`: owner OR shared(any level) OR public
- `create($user)`: any authenticated user
- `update($user, $issue)`: owner OR shared(edit)
- `comment($user, $issue)`: owner OR shared(comment or edit)
- `delete($user, $issue)`: owner only
- `share($user, $issue)`: owner only
- `restore($user, $issue)`: owner only

**CommentPolicy:**
- `create($user, $issue)`: delegates to IssuePolicy::comment

### Policy Registration
- Register in AuthServiceProvider (or auto-discovery)
- Apply in controllers via `$this->authorize()` or middleware

### Scopes (on Issue model)
- `scopeAccessibleBy($query, $user)`: returns issues where user is owner OR shared OR public
- Used in list endpoint to filter correctly

## Tests Required
Integration:
- I-06 (partial): verify private issue not accessible by unshared user → 403
- I-18 (partial): verify user only sees own + shared + public

Feature:
- Owner can view/update/delete own issue
- Shared(view) can view but not update → 403
- Shared(comment) can view and comment but not update → 403
- Shared(edit) can view, comment, and update
- Public issue: any user can view, cannot comment without share
- Non-owner cannot delete → 403
- Non-owner cannot share → 403
- Unauthenticated → 401 on all endpoints
