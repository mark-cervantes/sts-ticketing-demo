## What To Build

Category CRUD for the dynamic DB-backed category system (ADR-006).

### Endpoints
| Method | Route            | Action                  |
| ------ | ---------------- | ----------------------- |
| GET    | /categories      | List all categories     |
| POST   | /categories      | Create new category     |
| DELETE | /categories/{id} | Delete (if no issues)   |

### Controller
**CategoryController** (thin):
- `index()` â€” return all categories, ordered by name
- `store(StoreCategoryRequest $request)` â€” create with auto-slug
- `destroy(Category $category)` â€” delete if no issues reference it

### Form Request
**StoreCategoryRequest**:
- `name`: required, string, trimmed, unique (case-insensitive via Rule::unique with raw expression)

### Business Rules
- Slug auto-generated from name (model boot `creating` event)
- Duplicate name (case-insensitive) â†’ 422
- Delete: if category has issues â†’ 409 with count ("Cannot delete: 5 issues use this category")
- Delete: if no issues â†’ 204

### Response
- List: array of { id, name, slug }
- Create: 201 with { id, name, slug }
- Delete: 204

### Routes

### Case-Insensitive Uniqueness â€” Critical Pitfall
- The DB constraint (`add_unique_to_categories_name` migration) is a standard PostgreSQL `UNIQUE` on `name` â€” this is **case-sensitive** at DB level
- Application-level case-insensitive check must use `Rule::unique('categories', 'name')` with a raw `whereRaw('LOWER(name) = ?', [strtolower($value)])` â€” no `citext` or `LOWER()` index exists
- PostgreSQL will catch exact-case duplicates at DB level, but "Bug Reports" vs "bug reports" will only be caught by the Form Request validation rule
- Test must cover: create "Bug Reports" â†’ create "bug reports" â†’ expect 422 (not a DB constraint error, but a validation error)

### Deletion Guard
- `destroy()` must check `$category->issues()->count()` before delete; if > 0 return `response()->json(['message' => "Cannot delete: {$count} issues use this category"], 409)`
- The FK on `issues.category_id` has `restrictOnDelete()` (see `create_issues_table` migration line 22), so the DB will also prevent orphan deletes â€” but the controller must return a 409 with count **before** hitting the FK constraint
- Use `$category->loadCount('issues')` or `$category->issues()->count()` â€” the `issues()` HasMany relationship already exists in `Category` model (line 79)
