# Task 10.03: Kanban Filtering (My Tickets) + Column-Wide Sorting

## Problem
Users can't quickly find their own tickets on a busy board. There's no way to filter by ownership or sort columns by urgency/recency. On a board with 50+ tickets, this makes the Kanban unusable for individual contributors.

## Solution

### Sidebar Filter: "My Tickets" Toggle
Add a prominent toggle in the existing filter sidebar:
```
┌─────────────────────────────┐
│  👤 MY TICKETS              │
│  [Show only mine]  ○ / ●   │
│                             │
│  ⊙ STATUS                  │
│  ...existing filters...     │
│                             │
│  ⚠ PRIORITY                │
│  ...existing filters...     │
└─────────────────────────────┘
```
- Toggle ON: only show tickets where `issue.user.id === auth.user.id`
- This is client-side filtering (data is already loaded with the user relation)
- Position: top of the sidebar, above STATUS filters (it's the most-used filter)

### Sidebar Filter: Assignee (Future-Ready)
For now, "My Tickets" is a simple owner toggle. The architecture should support a future "Assigned to" dropdown when we add issue assignment.

### Column-Wide Sorting
Each column header gets a sort dropdown (or cycle button):
```
┌──────────────────────────────────┐
│ Open (12)         [↕ Priority]   │
├──────────────────────────────────┤
│  Critical tickets first...       │
│  High tickets next...            │
│  Medium...                       │
│  Low...                          │
└──────────────────────────────────┘
```

Sort options:
- **Priority** (default): Critical → High → Medium → Low
- **Newest first**: by `created_at` descending
- **Recently updated**: by `updated_at` descending
- **Oldest first**: by `created_at` ascending

Sorting is:
- Per-column (each column can have a different sort)
- Client-side (all data is already loaded)
- Persisted in localStorage per column slug
